Перевірка версій Node.js та npm

1.node --version— Перевіряє встановлену версію Node.js;
2.npm --version— Перевіряє встановлену версію npm.

Ініціалізація проекту

1.npm init— Запускає майстер створення package.json;
2.npm init -y— Створює package.json з налаштуваннями за замовчуванням.

Налаштування конфігурації npm

1.npm config set init-author-name "YOUR_NAME"— Встановлює ім'я автора;
2.npm config set init-author-email "YOUR_EMAIL"— Встановлює email автора;
3.npm config list— Показує поточну конфігурацію npm;
4.npm config get <key>— Отримує значення конкретного налаштування.

Встановлення пакетів

1.npm install <package-name>— Встановлює пакет локально;
2.npm i <package-name>— Скорочена версія команди;
3.npm install -g <package-name>— Встановлює пакет глобально;
4.npm install <package-name> --save-dev— Встановлює пакет як dev-залежність;
5.npm install <package-name> --save-prod— Встановлює пакет як основну залежність.

Видалення пакетів

1.npm uninstall <package-name>— Видаляє пакет;
2.npm r <package-name>— Скорочена версія команди;
3.npm uninstall -g <package-name>— Видаляє глобально встановлений пакет.

Оновлення пакетів

1.npm update— Оновлює всі пакети до minor/patch версій;
2.npm update <package-name>— Оновлює конкретний пакет;
3.npm outdated— Показує застарілі пакети;
4.npm install <package-name>@latest— Встановлює найновішу версію пакета.

Очищення кешу

1.npm cache clean --force— Очищає кеш npm;
2.npm cache verify— Перевіряє цілісність кешу.

Робота з директоріями

1.cd ~/projects— Перехід до папки "projects" у домашній директорії.

Запуск скриптів

1.npm start— Запускає скрипт start з package.json;
2.npm run <script-name>— Запускає користувацький скрипт з package.json;
3.npm run dev— Запускає dev-сервер.

Зупинка сервера або процесу

1.Ctrl + C— Для користувачів Windows;
2.Control + C— Для користувачів MacOS і Linux.

Управління залежностями

1.npm list— Показує встановлені пакети;
2.npm list -g --depth=0— Показує глобально встановлені пакети;
3.npm ls <package-name>— Показує інформацію про конкретний пакет.

Повторна установка залежностей

1.rm -rf node_modules package-lock.json && npm install— Видаляє папку node_modules і package-lock.json, а потім перевстановлює всі залежності;
2.npm ci— Встановлює залежності строго за package-lock.json.
